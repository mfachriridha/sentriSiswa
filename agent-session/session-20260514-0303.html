SentriSiswa — Register Validation Fix

Date
2026-05-14 03:03
Context
Registration page "Akun Demo" data not validating

Problem

User copy-pastes demo NIP/NIS from "Akun Demo" box on register page. Clicking "Validasi Data" shows "Data Tidak Ditemukan" or "Gagal Terhubung ke Server".

Root Cause 1
CSRF token mismatch (HTTP 419) — validation endpoints were not excluded from ValidateCsrfToken middleware. The X-CSRF-TOKEN header approach was rejected by Laravel's PreventRequestForgery middleware.
Root Cause 2
Regex validation ^\d+$ ran BEFORE whitespace cleaning. Copy-paste includes invisible whitespace/characters → regex fails → 422 validation error → returns HTML error page → JS r.json() fails → catch block triggers "Gagal terhubung ke server".

Fixes Applied

  1. CSRF Exclusion — bootstrap/app.php 
    $middleware->validateCsrfTokens(except: [
    'auth/register/validate-nip',
    'auth/register/validate-nis',
]);
  2. Clean Input Before Validation — RegisterController.php

    Replaced regex validation with manual cleaning: preg_replace('/[^\d]/', '', $input) strips ALL non-digit characters (spaces, zero-width chars, etc.) BEFORE querying database.

    Applied to all 4 methods: validateNip, validateNis, register (guru branch), register (siswa branch).

  3. Improved JS Error Handling — register.blade.php

    Added response.ok check before r.json(). Logs actual error message to console. Shows server error text instead of generic "Gagal terhubung ke server".
